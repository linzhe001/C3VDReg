# C3VDReg Dataset and Benchmark Paper Rewrite Plan

## Goal

把 `BMVC2026_Linzhe/bmvc_c3vd_benchmark_draft.tex` 从当前的
“C3VDReg-Bench + DPG-HPT/LLM setup” 改成一篇 dataset/benchmark paper：

> C3VDReg: a frame-wise dataset and benchmark for colonoscopic point-cloud registration.

主线只保留三件事：

1. C3VDReg 如何从 C3VD depth maps、camera poses、CT meshes 构造 frame-wise registration pairs。
2. 固定 benchmark contract 如何支持公平比较。
3. 当前 public registration baselines 在 colonoscopic tubular anatomy 上为什么失败。

不再把 DPG-HPT、LLM、agent workflow 作为论文贡献或主文内容。

## Main Positioning

推荐标题：

`C3VDReg: A Frame-wise Dataset and Benchmark for Colonoscopic Point Cloud Registration`

一句话定位：

`C3VDReg converts C3VD depth maps, camera poses, and CT meshes into viewpoint-matched partial-to-partial point-cloud registration pairs, with fixed splits, metrics, and baseline results for evaluating frame-wise colonoscopic registration.`

## MambaNetLK Style To Reuse

主要参考 `MambaNetLK_IPCAI/sn-article.tex` 的表达方式，而不是当前 BMVC 草稿的 HPT 叙事。

### Abstract Style

按医学论文式逻辑写：

- `Purpose`: Accurate frame-wise localization is important for colonoscopic navigation, where video-depth-derived local observations must be aligned with CT-derived anatomy.
- `Methods`: Introduce C3VDReg; source clouds from depth reprojection; target clouds from matched-pose CT-mesh raycasting; fixed split, perturbation, point budget, pose convention, and metrics.
- `Results`: Public baselines remain unreliable; GeoTransformer reaches only 17.43 percent RR@5 and 35.63 percent RR@10; failures are dominated by translation ambiguity despite high overlap.
- `Conclusion`: C3VDReg is a controlled intermediate benchmark, limited to rigid, viewpoint-matched partial-to-partial registration.

### Introduction Style

三段即可：

1. Clinical need: colonoscopy is frame-wise, local, and camera-centric; navigation requires localizing each observation on stable anatomy.
2. Gap: C3VD and related resources provide useful assets, but not fixed frame-wise point-cloud registration pairs, splits, perturbations, pose convention, metrics, and baselines.
3. Contribution: C3VDReg dataset, benchmark contract, and failure analysis showing translation ambiguity and misleading local residuals.

### Dataset Style

复用 MambaNetLK 的 `The C3VD-Raycasting-10k Dataset` 思路：

- C3VD provides videos, depth maps, camera poses, and CT-derived meshes.
- Source point cloud `P_S`: generated by depth-map reprojection using camera intrinsics and pose.
- Target point cloud `P_T`: generated by placing a virtual camera at the matched pose and raycasting the CT mesh.
- Nearest ray-mesh intersection is retained to model the visible anatomical surface.
- The pair is viewpoint-matched but not measurement-identical, so it remains partial-to-partial and density-mismatched.

必须主动承认：

> The raycast target is an oracle visible CT surface. C3VDReg isolates frame-wise rigid alignment after visible-region selection; it does not solve full local-to-global retrieval.

## Core Argument

全文保持这条逻辑：

1. Clinical colonoscopy observes only local surface patches frame by frame.
2. The clinically desired endpoint is local-to-global localization against a full anatomical reference.
3. Direct single-frame local-to-full-CT registration is currently too ambiguous for standard point-cloud registration methods because overlap is tiny and tubular regions are repetitive.
4. C3VDReg uses matched-pose raycasting to provide an oracle visible CT target.
5. This creates a controlled partial-to-partial benchmark that isolates the alignment subproblem.
6. Even under this simplified setting, current baselines remain unreliable.

Use this wording:

> C3VDReg should be interpreted as a controlled intermediate benchmark toward clinical local-to-global colonoscopic registration.

Avoid:

- `solves clinical navigation`
- `clinically deployable`
- `enables navigation`
- any claim that raycast partial-to-partial is the final clinical problem

## Contributions

Use three contributions only:

1. `C3VDReg Dataset`
   - 10,015 C3VD-derived viewpoint-matched partial-to-partial point-cloud pairs.
   - Source from depth-map reprojection.
   - Target from matched-pose CT-mesh raycasting.
   - Includes pair manifests, fixed splits, generation/evaluation code, subject to upstream C3VD terms.

2. `Benchmark Contract`
   - Fixed train/validation/test split.
   - Held-out test scenes.
   - 8,192 source and target points.
   - Source-only R25--90deg/T100--500mm perturbation.
   - Source-to-target pose convention.
   - Metrics: RRE, RTE, RR@5, RR@10, marginal R<=5/T<=5, visible NN, trimmed Chamfer, runtime/memory.

3. `Failure Analysis`
   - Current baselines remain far from reliable.
   - Translation, not rotation, is the main bottleneck.
   - High ground-truth overlap does not guarantee correct pose.
   - Local residual metrics can hide clinically important pose failures.

### 5. Technical Validation and Benchmark Results

Use existing results, but explain them as dataset validation.

1. `Quantitative analysis`
   - GeoTransformer is strongest but only 17.43 percent RR@5 and 35.63 percent RR@10.
   - C3VDReg is unsaturated.

2. `Translation bottleneck`
   - GeoTransformer R<=5: 56.18 percent.
   - GeoTransformer T<=5: 18.15 percent.
   - Strict RR@5 is limited by translation.

3. `Scene-level generalization`
   - Scene-level RR differs strongly across held-out anatomy.

4. `Overlap analysis`
   - GT overlap quantiles are all high: 74.3--93.1 percent.
   - RR@5 is nonmonotonic across overlap quantiles.
   - Failure is not simply missing shared surface.

5. `Qualitative analysis`
   - Show GeoTransformer success and plausible-but-wrong failure.
   - Emphasize local residual can look good while global pose is wrong.

### 6. Discussion and Limitations

Must include:

- Clinical relevance, stated conservatively: frame-wise localization can support anatomy-referenced navigation, coverage assessment, revisited-region detection, and follow-up localization, but C3VDReg does not validate clinical deployment.
- C3VDReg complements existing colonoscopy datasets by providing fixed frame-wise registration pairs and evaluation.
- The benchmark is rigid, viewpoint-matched, and partial-to-partial.
- The raycast target is an oracle visible CT target.
- It does not solve full local-to-global retrieval.
- It does not model nonrigid deformation, debris, fluid, tissue motion, insufflation changes, or full clinical workflow.
- Future work: full local-to-global localization, sequence constraints, anatomical priors, uncertainty-aware registration, nonrigid/in vivo validation.

### 7. Conclusion

Only say:

- We introduce C3VDReg, a frame-wise dataset and benchmark for colonoscopic point-cloud registration.
- Current public baselines remain unreliable under strict pose metrics.
- Tubular anatomy causes translation ambiguity and residual-metric failure cases.
- C3VDReg provides a controlled intermediate benchmark toward clinical colonoscopic localization.
